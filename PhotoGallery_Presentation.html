Why Crypto Random?

Uses crypto.getRandomValues() instead of Math.random() - cryptographically secure randomness prevents predictability attacks.

Example output: kJ9$mN2#pQ5@vX9&

// Generate secure random password const generateSecurePassword = () => { const charset = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ' + 'abcdefghijklmnopqrstuvwxyz' + '0123456789!@#$%^&*'; let password = ''; // Crypto random - NOT Math.random() const randomValues = new Uint32Array(16); crypto.getRandomValues(randomValues); for (let i = 0; i < 16; i++) { password += charset[ randomValues[i] % charset.length ]; } return password; }; // Result: kJ9$mN2#pQ5@vX9& // 16 chars, guaranteed complexity

Argon2 Password Hashing

Winner of Password Hashing Competition (2015)

  • Argon2id variant: Hybrid mode - GPU resistant + side-channel resistant
  • 64 MB memory cost: Makes GPU attacks expensive
  • 3 iterations: ~500ms per hash (user-friendly, attacker-hostile)
  • Unique salt: Every password gets random 16-byte salt

Why Memory-Hard?

MD5: ~1KB memory → GPU can hash 8 million in parallel

Argon2: 64MB memory → GPU can hash only 125 in parallel

Result: 64,000× slower for attackers

// Hash configuration await argon2.hash(password, { type: argon2.argon2id, // Memory cost: 64 MB per hash memoryCost: 2 ** 16, // Time cost: 3 iterations timeCost: 3, // Single thread parallelism: 1 }); // Output format: $argon2id$v=19$m=65536,t=3,p=1$[salt]$[hash] ↑ ↑ ↑ ↑ ↑ │ │ │ │ └─ Hash value │ │ │ └───────── Random salt │ │ └────────────────────────── Parameters │ └────────────────────────────── Version └──────────────────────────────────────── Algorithm // Unique salt = 2^128 possibilities // Rainbow tables: IMPOSSIBLE

Database Structure
